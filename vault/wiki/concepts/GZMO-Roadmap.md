# GZMO Roadmap

## Vision
GZMO evolves toward full autonomous sovereignty: an edge AI daemon that maintains, improves, and documents itself without human intervention.

## Completed ✅
- [x] Lorenz attractor-driven Chaos Engine (PulseLoop at 174 BPM)
- [x] Allostatic stress system (tension/energy homeostasis)
- [x] Vault RAG with nomic-embed-text (2000+ chunks indexed)
- [x] Task routing (think/search/chain actions)
- [x] Dream Engine (autonomous distillation, 30min cycle)
- [x] Self-Ask Engine (gap detective, contradiction scanner, spaced repetition)
- [x] Thought Cabinet crystallization pipeline
- [x] Episodic memory for cross-task continuity
- [x] Bun native I/O migration (zero sync I/O)
- [x] Hermes3:8b model integration (superior agentic reasoning)
- [x] Wiki Engine (autonomous knowledge consolidation)

## In Progress 🔄
- [ ] Wiki Engine self-documentation (reading own source code)
- [ ] Full autonomous knowledge loop verification
- [ ] Stress test expansion to cover Wiki Engine

## Planned 📋
- [ ] Hardware Sonification Pipeline (HSP) integration with Chaos Engine
- [ ] Telegram notification for critical events (opt-in, no API coupling)
- [ ] systemd service with auto-restart and watchdog
- [ ] Model hot-swap capability (switch models based on task complexity)
- [ ] Vault deduplication and quality scoring
- [ ] Cross-session knowledge graph (linking wiki pages)

## Design Principles
1. **Sovereignty**: No cloud dependencies. Everything runs locally.
2. **Filesystem-first**: Obsidian Vault is the single source of truth.
3. **Anti-sedation**: The allostatic system prevents the daemon from going idle.
4. **Graceful degradation**: If Ollama dies, the heartbeat keeps pulsing.
5. **8 GB VRAM constraint**: Every optimization must respect this hard limit.
